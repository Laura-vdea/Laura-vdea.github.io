이번 주 수업에서는 임베디드 개발에 필요한 라즈베리파이와 GPIO 제어 기술을 학습했다. 이를 통해 하드웨어와 소프트웨어가 상호작용하는 구조에 대해 더 깊은 이해를 할 수 있었다.

## 1. WiringPi 설치 및 실습

먼저라즈베리파이에서 GPIO 제어를 하기 위해 WiringPi를 설치하였다. 실습을 하기 전에 특정 라이브러리가 필요하였기 때문에 아래의 명령어를 통해 설치를 진행했다.

```
$ sudo wget https://github.com/WiringPi/WiringPi/releases/download/3.10/wiringpi_3.10_arm64.deb
$ sudo dpkg -i wiringpi_3.10_arm64.deb
```

설치 후, 다음 명령어로 GPIO 핀 상태를 확인하고 LED를 제어하는 실습을 진행했다.

```
$ gpio readall
$ gpio -g write 18 1
$ gpio -g write 18 0
```

gpio -g write 18 1 은 전구를 킬 수 있는 명령어고,
반대로
gpio -g write 18 0은 전구를 끌 수 있는 명령어다.
먼저 이 두가지의 명령어로 본인이 회로도를 보고 잘 꽂았는지 확인을 하고 코드를 실습하여 진행을 하면 된다.

## 2. 코드 실습

### 2.1 LED 제어 코드 (wiringled.c)

```
#include <wiringPi.h>
#include <stdio.h>
#include <stdlib.h>

int ledControl(int gpio) {
    int i;
    pinMode(gpio, OUTPUT);

    for(i = 0; i<5; i++) {
        digitalWrite(gpio, HIGH);
        delay(1000);
        digitalWrite(gpio, LOW);
        delay(1000);
    }

    return (0);
}

int main(int argc, char** argv) {
    int gno;
    if(argc<2) {
        printf("Usage : %s GPIO_NO\n", argv[0]);
        return -1;
    }
    gno = atoi(argv[1]);
    wiringPiSetup();
    ledControl(gno);

    return (0);
}
```

위 코드를 통해 GPIO 핀을 통해 LED를 켜고 끄는 작업을 1초 간격으로 5번 반복하여 제어할 수 있다.
개인적으로 이 코드가 임베디드 수업 중 가장 재미있던 시간이 아닐까 싶다. 교실에 울러퍼지는 음악소리가 교실의 분위기를 좋게 만들어줬다~ㅋㅋ

### 2.2 음악 연주 코드 (wiringmusic.c)

```
#include <wiringPi.h>
#include <softTone.h>

#define SPKR 6
#define TOTAL 32

int notes[] = {
    391, 391, 440, 440, 391, 391, 329, 329,
    293, 293, 261, 261, 293, 329, 349, 349,
    391, 391, 440, 440, 391, 391, 329, 329,
    293, 293, 261, 261, 293, 329, 349, 349
};

int musicPlay() {
    int i;
    softToneCreate(SPKR);

    for(i=0; i<TOTAL; ++i) {
        softToneWrite(SPKR, notes[i]);
        delay(280);
    }

    return (0);
}

int main() {
    wiringPiSetup();
    musicPlay();
    return (0);
}
```

이 코드는 softTone을 사용하여 간단한 멜로디를 출력하는 프로그램이다. GPIO 6번 핀을 스피커에 연결하여 음악을 연주할 수 있었다.

## 3. 베다만의 취업 특강

### 특강에 대한 느낀 점

특강에 대한 걸 자세히 풀면 한화비전의 대외비 문제로 간단히 느낀 점만 적겠습니다.

##### 1. 임베디드 시스템 개발의 특성 특강 중 한화비전의 카메라 백엔드 시스템에 대한 내용을 듣고, 임베디드 시스템 개발이 하드웨어와 소프트웨어의 밀접한 상호작용을 요구한다는 점을 다시금 깨달았습니다.

시스템의 성능 최적화와 하드웨어 자원의 효율적 사용이 중요한 임베디드 환경에서는 메모리 관리, 전력 효율, 실시간 데이터 처리와 같은 요소들도 간과할 수 없지만, 백엔드에서 이를 API로 어떻게 받아서 처리하는 기술을 보고 어떻게 협업을 해야할 지 감을 잡을 수 있던 시간이 되어주었습니다.
결국은 브라우저에서 볼 수 있는 것이 사용자 편의성 때문에 좋기 때문에 모든 건 서버에서 빨리 `try`되어야해서 임베디드 분야도 `sils`라고 서버에서 빠릴 검증한 후에 넘어가는 것으로 방향이 되어있는 것으로 알고 있기 때문입니다.

##### 2. PM과 협업의 중요성 PM과의 협업 방식을 다룬 특강에서는 프로젝트 관리자가 개발자들과 긴밀히 협력하여, 일정 관리와 자원 배분을 조율하는 과정에서의 주요 역할을 배울 수 있었습니다.

임베디드 개발 과정에서는 하드웨어 제약과 소프트웨어 개발 주기를 균형 있게 맞추는 것이 중요하긴 하지만 프로젝트의 성공을 위해서는 역시 PM의 역할이 매우 중요하다는 것을 다시 한번 느낄 수 있었습니다.
Project Manager와 Product Manager의 차이에 대해서도 배울 수 있었으며, 기술자와 PM 간의 원활한 의사소통과 협업은 프로젝트 성공의 열쇠라고 느낄 수 있었습니다.

##### 3. 실무에서의 직무 특강 이번 특강을 통해 실무에서 요구되는 스킬셋을 재확인할 수 있었습니다.

자소서나 포트폴리오에 자신만의 경험과 프로젝트를 잘 녹여내기 위해 질문을 하였고, 어떻게 쓰고 기술면접에 대하여 어떻게 대비하여야할 지 감을 잡을 수 있던 소중한 시간이 되어주었습니다. 현재 취업 트렌드를 반영한 자소서 작성법과 포트폴리오 구성에 대한 실질적인 조언이 인상 깊었습니다  
 들은 조언들을 바탕으로 현재 진행 중인 프로젝트에 대한 정리를 잘 준비하고, 취업 준비 과정에서 효과적으로 활용해야겠다는 다짐을 하게 되었습니다.
